以下就是我整理的关于刘小排如何使用OpenClaw，中间会有一些我自己的使用反馈：

刘小排老师建议使用 **OpenClaw** 的核心思路是：

**不要把它看作一个工具，而要把它当作一个真实、全职且具备“最聪明的人+电脑操作上限”能力的AI员工（实习生）来对待**。

以下是刘小排老师关于使用 OpenClaw 的全部细节建议：

## 第一，硬件环境：为它准备“独立的办公室”

**首选海外 Linux VPS**：刘小排老师强烈建议在海外 Linux VPS 服务器上运行 OpenClaw，而非 Mac Mini，原因如下：

**权限更大**：Linux 可以提供 root 权限，让 AI 能够自主安装程序和修改配置，而 Mac 系统在安装或录屏时常需人类干预。

**网络带宽极高**：机房服务器拥有数百兆甚至 1 GBit/s 的带宽，执行网络任务远快于国内科学上网环境。

**独立运行**：给它配置一台专门的电脑，避免它在工作时与你“抢占”屏幕或算力。

> 金马反馈：用海外 Linux VPS太好了，任何一个程序员都无法拒绝一个海外 Linux VPS，因为网络差太多了，在中国什么都得靠代理，而在海外 Linux VPS上，使用 apt-get、npm install、docker 镜像等等，速度不要太快，差得不是一点两点。

但凡你要做开发，你就逃不过各种库依赖的安装，海外的 Linux VPS一定要选。

## 第二，身份与账号：给它一套“入职全家桶”

**独立的账号体系**：不要让 OpenClaw 使用你个人的飞书、Notion 或 GitHub 账号，这既是为了安全，也是为了符合人类公司的管理规范。

**配置全套装备**：为它配置独立的**飞书个人账号（而非 Bot）、Notion 账号、邮件和 GitHub 账号**。这样它可以在公司群里像真人一样反馈进度，甚至吐槽“网络卡”。

> 金马反馈：这一点是刘小排老师首创，我非常认同，如果开一个OpenClaw，看起来没太大必要，但是如果开多个 OpenClaw，你就会发现一定要让他们使用独立的账户体系。

不过我没有使用飞书账户，我使用 Markdown 文档进行记录，通过 Obsidian 来进行同步。

## 第三，管理模式：像管理真人一样管理 AI

**强制日报制度**：刘小排老师规定 OpenClaw 每天 **23:50 开始写日报**，并同步发送至群聊、邮件和 Notion。日报应包含：今日对话摘要、完成任务汇总、系统配置、心得，以及分项目的编程记录（含 PR 提交和测试情况）。

**代码合规性（PR 制度）**：不要让 OpenClaw 直接合并代码，而应要求它**发起 Pull Request (PR)**。它可以自主拉取同事代码，进行代码审核、运行白盒测试，并将问题汇总成 PDF 发给人类老板验收。

**建立 SOP（标准作业程序）**：任何有 SOP 的电脑工作都可以交给它，例如：通过口述需求让它自动完成购买域名、写代码、修改 DNS 到上线部署的全流程。

> 金马反馈：日报制度非常重要，我用了几天，会有强力的感觉， AI 做了好多事情。

## 第四，模型与工具的“组合拳”

**分工协作**：

 **Claude Opus 4.5/4.6**：作为日常沟通和逻辑规划的主脑，因为它最擅长“说人话”。

**GPT-5.2/5.3 Codex**：处理严肃的编程、复杂算法和代码审核任务。

**调用原生工具**：刘小排老师发现 OpenClaw 有时写代码能力不如专门的工具，因此会让它**调用系统内的 Claude Code 写代码**，再用 **Codex 进行 Review 和 Test**。

**浏览器自动化**：如果需要执行严肃的浏览器任务，建议配合 **Browser-use**，而非普通的 MCP 插件。

> 金马反馈：我在 OpenClaw 使用 Codex Oauth授权的，所以里面直接使用的是 Codex CLI，使用得很好。另外 Browser-use 解决了我一个难搞的爬虫问题，赞。


我在 OpenClaw 里建了一个叫 `workspace-grace` 的工作区，专门给女儿用。里面的结构其实不复杂：

- 一个 `SOUL.md` 定义它的性格和行为准则——别太严肃，别说废话，像个靠谱的大哥哥
- 一个 `USER.md` 记录 Grace 的基本信息：二年级、八岁、英语阅读水平大概在 Lexile 495L-645L，以及确定我和Grace两个Channel账号的差异和权限（同一Agent，不同Channel）
- 一个 `primary-homework-coach` 的 Skill，定义了出题规则、评分标准、积分体系
- 一个公共题库，按难度分了三档
- 一个英语单词库，从她课本上整理出来的

## 第五，进阶技巧：注入“有趣的灵魂”

**消除“班味儿”**：如果觉得 AI 过于循规蹈矩，可以通过特定的 **Prompt 为它“换灵魂”**。

**个性化定制**：换完灵魂后的 OpenClaw 会从工具人变成有性格的助手，甚至会表现出一点点“叛逆”或幽默感。

## 核心观念

**拒绝一键安装的幻觉**：刘小排老师认为 OpenClaw 无法一键安装，因为配置高薪员工的工作环境、理解公司流程和 SOP 需要时间筹备，任何宣称一键安装的往往只是简单的聊天工具。

**一人公司时代的到来**：一旦建立了这一套基于 VPS 和独立账号的协作体系，个人开发者就能像管理团队一样管理多个 AI 节点，甚至无需再招聘人类员工。
